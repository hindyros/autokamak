# Example workspace contract

Everything under `examples/` is a **runnable-without-LLM artifact**. Agents (via `plan_execute_feedback`) generate new examples; humans can run them directly.

## Layout contract

A valid workspace has these files at the root:

```
examples/<name>/
├── README.md            # what this example does, how to run it
├── run_*.py             # single Python entry point (argparse or positional config)
├── *_config.yaml        # the physics/dataset/AutoML YAML the runner reads
├── outputs/             # sink for generated artifacts (gitignored)
└── (optional)
    ├── ursa/            # symlink materialized by the agent runner
    └── OpenFUSIONToolkit/
```

**Anti-pattern:** never commit anything under `outputs/`. It's gitignored for a reason.

## Reference template

Use `examples/config_driven_equilibrium/` as the reference. Its `run_equilibrium_from_config.py` is the canonical shape:

1. `argparse` positional `config` arg + optional `--validate-only`.
2. `load_config(path)` → returns a validated dict (uses `EquilibriumConfig.from_yaml`).
3. `build_mesh_and_boundary(cfg)` → wraps `autotokamak.core.geometry.build_mesh_from_config`.
4. `setup_and_solve(cfg, mesh_pts, mesh_lc, mesh_reg, lcfs)` → wraps `autotokamak.core.solver.solve_equilibrium`.
5. `save_case_outputs(cfg, gs_mesh, gs, ...)` → writes `summary.json`, `summary.yaml`, `raw_arrays.npz`, `mesh_*.png`, `psi_*.png`, `config_used.yaml` under `outputs/<case_slug>_<hash>/`.

The runner deliberately does NOT import from `autotokamak.eval.*` or `autotokamak.surrogate.*` — those are Phase-2+ concerns.

## When adding a new example

1. Start from `assets/templates/example_workspace/` — copy it into `examples/<new_name>/`.
2. Import from `autotokamak.core` — don't reinvent geometry, solver, or IO.
3. Config-drive the physics — don't hardcode `mesh_dx`, `order`, or targets.
4. Write outputs under `outputs/` — never at the workspace root.
5. Emit a `summary.json` at the top of `outputs/<case>/` so downstream scoring can find it deterministically.
6. If the example is meant to be **agent-generated**, add a `problem:` prompt to `src/autotokamak/agent/prompts/<name>.yaml` and point its `workspace:` at the new dir.

## Do NOT edit generated workspace runners

`examples/surrogate_automl/run.py` and anything under `experiments/<run_id>/` are agent output. If they contain a bug, **fix the platform** (a prompt, a runner, a scorer, or a core function) and regenerate. See `feedback_dont_edit_agent_output` — this is a load-bearing project rule.
