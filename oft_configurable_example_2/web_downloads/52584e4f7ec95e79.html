    Release v1.0.0-beta7 Latest
    Latest

    Updated beta release of the Open FUSION Toolkit.

    Important

    This release introduces breaking changes for the OFT Python interface as a initialization of the environment has changed. This should require only minimal code changes. See the documentation for more information and the TokaMaker examples and ThinCurr examples for demonstrations.

    Note

    The beta status reflects possible issues that may have been introduced in the refactoring for open-sourcing. See Milestone #1 as a tracker for progress to a full v1.0.0 release.

    New Features

    • TokaMaker: Add option to update bounds when manually setting psi [#177]
    • TokaMaker: Reenable vacuum solves through normal .solve() method [#178]
    • TokaMaker: Add COCOS=2 option to .save_eqdsk() [#184]
    • TokaMaker: Add support for using Cubit as the meshing backend [#186]
    • TokaMaker: Add support for Jphi and P' formulation [#188]
    • TokaMaker: Reenable maxima filtering in O-/X-point detection [#189]
    • TokaMaker: Add experimental support for mirror equilibria [#193]
    • TokaMaker: Remove equilibrium solver constraints if set before reconstruction [#196]
    • TokaMaker: Prevent the usage of more threads than allocated space [#207]
    • TokaMaker: Add Isoflux/NSTX examples [#199]
    • TokaMaker: Add pulse design example [#200]
    • TokaMaker: Improve Jphi profile handling with non-zero edge values [#230]
    • TokaMaker: updated solve_with_bootstrap() and ITER H-mode example [#195]
    • ThinCurr: Add force calculation example [#181]
    • ThinCurr: Improve support for region-dependent resistivity via Exodus attributes [#219]
    • ThinCurr: Add a class that place & analyze Mirnov sensors on flux surface [#176]
    • ThinCurr: Added XML coil construction support with example notebook [#225]
    • ThinCurr: Deprecate ThinCurr executables [#238]
    • Add logos to repo, website, and documentation [#180]
    • Add build_libs.py support for statically building more libraries [#211]
    • Bump supported and default PETSc versions [#214]
    • Handle MPI Fortran 2008 module support with PETSc [#222]
    • Add support for custom environment setup with copilot agents [#242]
    • Add test runs for examples as part of CI actions [#241]

    Fixes

    • TokaMaker: Fix bugs in EFIT readers introduced in a4a397c [#183]
    • TokaMaker: Fix bugs in get_jtor_plasma() and psi_init() [#185]
    • TokaMaker: Fix some performance regressions introduced in 2d8b3a8 [#194]
    • TokaMaker: Fix invalid saddle scale filtering [#209]
    • TokaMaker: Fix bug in get_conductor_source #227
    • TokaMaker: Fix Vloop calculations with non-inductive current [#231]
    • TokaMaker: Fix bug in calc_loopvoltage() when no non-inductive current profile is used [#240]
    • ThinCurr: Fix bugs in B-field operator construction and hole location [#179]
    • ThinCurr: Fix region filtering in ThinCurr force script [#190]
    • ThinCurr: Fix Cubit journal file to define cylinder jumper nodeset [#215]
    • ThinCurr: Fix performance bug with extrapolation in time-dependent runs [#226]
    • ThinCurr: Update the default Mirnov sensor's flux loop side length for denoising the signal [#229]
    • ThinCurr: Fix format size/string size mismatch in tw_load_coils() [#233]
    • Disable executable stack by default, fixes #201
    • Fix library discovery when packaging on Linux, fixes #218
    • Fix runtime issues introduced with Intel oneAPI 2025.3 [#221]
    • Fix warnings with pedantic compiler settings [#191]

    New Contributors

    • @benjysmith made their first contribution in #207
    • @ansonbraun made their first contribution in #215
    • @nelsonand made their first contribution in #199
    • @ThomasWangAPAM made their first contribution in #176
    • @JamieXia made their first contribution in #225

    v1.0.0-beta6

    Updated beta release of the Open FUSION Toolkit.

    Important

    This release introduces breaking changes for the OFT Python interface as a initialization of the environment has changed. This should require only minimal code changes. See the documentation for more information and the TokaMaker examples and ThinCurr examples for demonstrations.

    Note

    The beta status reflects possible issues that may have been introduced in the refactoring for open-sourcing. See Milestone #1 as a tracker for progress to a full v1.0.0 release.

    • TokaMaker: Add optional damping term for linear calculations to limit maximum growth rate [#132]
    • TokaMaker: Add support for specifying a non-zero LCFS pressure in EQDSK output [#135]
    • TokaMaker: Added current distribution coils [#127]
    • TokaMaker: Remove restriction on single TokaMaker instance [#141]

    • TokaMaker: Add K File and MHDIN Readers [#166]

    • ThinCurr: Add support for general voltage sources in time-dependent solves [#119]
    • ThinCurr: Add class to handling periodic toroid models [#124]
    • ThinCurr: Add support for simple mesh specification in Python [#128]
    • ThinCurr: HDF5 coil specification support [#131]
    • ThinCurr: Always create jumpers for hole elements [#167]
    • MUG: Add GEM challenge example (thin 3D) [#144]
    • Marklin: Remove restriction on single Marklin instance [#142]

    • Core: Restructure plotting to be more scalable [#123]

    v1.0.0-beta5 Pre-release
    Pre-release

    Updated beta release of the Open FUSION Toolkit.

    Important

    This release introduces breaking changes for ThinCurr models as the Python interface has changed significantly (including a change in units). See the documentation for more information and the ThinCurr examples for demonstrations.

    Note

    An internal copy of triangle has been added to OFT, so installation of the triangle python package is no longer required to generate TokaMaker meshes.

    Note

    The beta status reflects possible issues that may have been introduced in the refactoring for open-sourcing. See Milestone #1 as a tracker for progress to a full v1.0.0 release.

    New features

    • TokaMaker: Add error handling to .save_eqdsk() (failed tracing) (#68)
    • TokaMaker: Improve O-point location and tracking (#72)
    • TokaMaker: Add geometric major and minor radius to output information (#85)
    • TokaMaker: Only recompute operator for quasi-static solves if time step changes (#84)
    • TokaMaker: Increase accessible size of case field in gEQDSK file I/O (#90)
    • TokaMaker: Add support for untruncated EQDSK output (#93)
    • TokaMaker: Add support for quadrature-based area integration (#99)
    • TokaMaker: Add internal copy of triangle (#102)
    • TokaMaker: Add support for general current source terms in vac_solve and init_psi (#108)
    • ThinCurr: Add functionality to enable model reduction (#45)
    • ThinCurr: Improve model reduction interface (#71)
    • ThinCurr: Use Hurwitz and Landreman limiting form of self-inductance for filamentary coils (#73)
    • ThinCurr: Add script to automatically computes holes/closures from mesh files (#94)
    • ThinCurr: Add support for current jumpers in Python interface (#98)
    • ThinCurr: Use SI units for matrices relating to Icoils (#101)
    • Marklin: Improve plotting support (#100)
    • Core: Add Ubuntu 24.04 runner to CI action (#81)
    • Core: Add test tracking support (codecov.io) to CI tests (#82)
    • 3rd Party: Add support for specifying the SDK path on macos (#80)
    • Update publications on website

    Fixes

    • TokaMaker: Correct sign of eddy currents computed from dpsi/dt, fixes #78 (#79)
    • TokaMaker: Fix behavior of limiters with multiply connected plasma domains (#92)
    • ThinCurr: Handle corners in hole definitions (#77)
    • ThinCurr: Fix mu0 error in various Icoil operators (introduced in a0d586b) (#102)
    • Core: Fix memory leak with repeated factorizations using UMFPACK (#70)
    • Core: Fix compile error on Windows (WSL) due to carriage returns in git output (#75)
    • Core: Fix GitHub actions workflows for macOS (#86)
    • Core: Fix various compiler warnings during build (#104)
    • 3rd Party: Fix invalid URLs in build_libs.py (#69)
    • 3rd party: Fix missing zlib link on some platforms (#96)

    • Core: Fix build_libs.py failures:
      • Fix various issues with build_libs.py builds on different platforms
      • Ensure METIS is built with -fPIC (#10)
    • Core: Fix calculation of optimal operator smoothing coefficients
    • Core: Avoid HDF5 errors with missing intermediate groups in hdf5_field_exist
    • Core: Fix bug in native LU destructor for SuperLU-DIST
